## 3. 后端（reverie/backend_server）

### 3.1 核心入口与地图执行

- [reverie.py](e:/generative_agents/reverie/backend_server/reverie.py)：仿真主循环、读写 movement/env、驱动 persona  
- [maze.py](e:/generative_agents/reverie/backend_server/maze.py)：读取 matrix 语义网格、维护 tile 事件  
- [path_finder.py](e:/generative_agents/reverie/backend_server/path_finder.py)：寻路

### 3.2 Agent核心

- [persona/persona.py](e:/generative_agents/reverie/backend_server/persona/persona.py)：认知主链路 `perceive -> retrieve -> plan -> reflect -> execute`

#### 认知模块

- [perceive.py](e:/generative_agents/reverie/backend_server/persona/cognitive_modules/perceive.py)：环境感知与事件入记忆  
- [retrieve.py](e:/generative_agents/reverie/backend_server/persona/cognitive_modules/retrieve.py)：记忆检索  
- [plan.py](e:/generative_agents/reverie/backend_server/persona/cognitive_modules/plan.py)：计划、反应、对话触发  
- [reflect.py](e:/generative_agents/reverie/backend_server/persona/cognitive_modules/reflect.py)：反思生成 thought  
- [execute.py](e:/generative_agents/reverie/backend_server/persona/cognitive_modules/execute.py)：把动作状态变成路径与下一步  
- [converse.py](e:/generative_agents/reverie/backend_server/persona/cognitive_modules/converse.py)：对话内容生成

#### 记忆结构

- [scratch.py](e:/generative_agents/reverie/backend_server/persona/memory_structures/scratch.py)：短期状态（act/chat/path）  
- [associative_memory.py](e:/generative_agents/reverie/backend_server/persona/memory_structures/associative_memory.py)：事件/思考/聊天记忆  
- [spatial_memory.py](e:/generative_agents/reverie/backend_server/persona/memory_structures/spatial_memory.py)：空间记忆

#### 模型调用与提示词

- [prompt_template/run_gpt_prompt.py](e:/generative_agents/reverie/backend_server/persona/prompt_template/run_gpt_prompt.py)：prompt 组装与输出清洗  
- [prompt_template/gpt_structure.py](e:/generative_agents/reverie/backend_server/persona/prompt_template/gpt_structure.py)：OpenAI 调用封装  
- `prompt_template/v1|v2|v3_ChatGPT/`：多版本提示词模板

## 4. 当前可运行链路（文件视角）

1. 前端写环境步进：`storage/<sim_code>/environment/<step>.json`（`process_environment`）  
2. 后端读取环境并驱动 `persona.move(...)`  
3. 后端写动作结果：`storage/<sim_code>/movement/<step>.json`  
4. 前端读 movement 更新地图角色（`update_environment` + Phaser）
